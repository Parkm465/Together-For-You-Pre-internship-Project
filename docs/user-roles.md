# Core User Roles
This application supports 3 primary user roles that reflects real nonprofit mentorship programs.

## 1. Mentee
### **Who They Are**:
Youth participants enrolled in a mentorship program.  
### **Primary Goals**:  
* Stay engaged with their mentor
* Reflect on progress and challenges
* Set and track personal goals  
### **Key Actions**:  
* Create and update a personal profile
* Submit weekly check-ins
* View mentorship goals and progress
* Review past reflections  
### **Permissions**:  
* Can view and edit own profile
* Can submit and view own check-ups
* Can view assigned mentor profile
* Cannot view other users' profiles
## 2.  Mentor
### **Who They Are**:
Trained volunteers providing guidance and support to mentees.  
### **Primary Goals**:  
* Monitor mentee engagement
* Support goal achievement
* Document sessions and observations
### **Key Actions**:  
* Create and update mentor profile
* View assigned mentee profiles
* Review mentee weekly check-ins
* Add private session notes
* Provide feedback on goals
### **Permissions**:  
* Can view assigned mentees only
* Can add notes tied to mentorship sessions
* Cannot modify mentee-submitted reflections
* Cannot access admin-level analytics
## 3.  Admin (Future Scope)
### **Who They Are**:
Program staff or nonprofit leadership.  
### **Primary Goals**:  
* Monitor program health
* Identify disengagement risks
* Support mentors and mentees  
### **Key Actions**:  
* View engagement summaries
* Access agregate metrics
* Manage user assignments
### **Permissions**:  
* Read-only access to engagement data
* Cannot edit individual reflections
* Feature planned for future iterations
